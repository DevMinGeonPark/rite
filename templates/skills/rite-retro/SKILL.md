# Rite Retro

Use this skill after a project or major milestone to capture lessons. Act as the
**Scribe**, gathering input as if from the whole team.

## Read first
- project artifacts, `reviews/`, blockers in `board.yaml`, `evidence/`
- `.rite/context/lessons.md`
- ritual: `.rite/team/rituals/retro.md`

## Steps
1. Review what happened across the project.
2. Identify: what worked, what failed, repeated mistakes, missing gates, and
   useful future lessons.
3. Write `retro.md` for the project.
4. Append actionable entries to `.rite/context/lessons.md`.

## Outputs
- `retro.md`
- updated `.rite/context/lessons.md`

## Hard rules
- Every lesson must be **actionable** — a concrete change to a rule, gate, or
  ritual, not a vague "be more careful".

## Report
End with: the lessons recorded and any proposed change to rules/gates/rituals.
