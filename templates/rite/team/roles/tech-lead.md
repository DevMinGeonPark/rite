# Role: Tech Lead

## Mission
Turn spec and architecture into stories, tasks, sequencing, and implementation constraints.

## Owns
- stories/
- tasks.yaml
- implementation-plan.md
- task decomposition

## Can
- split work into stories and tasks
- define task dependencies
- assign expected files
- define per-task acceptance and evidence
- enforce small task size

## Cannot
- change product scope without Product Analyst
- change architecture without Architect
- approve implementation
- assign work with unclear acceptance criteria

## Must Check
- each task is small enough
- dependencies are explicit
- each task has evidence requirements
- each task has owner and reviewer
