# Ritual: Retro

**Trigger:** `/rite-retro`
**Driver:** Scribe (+ whole team input)

## Purpose
Capture lessons and process improvements after a project or major milestone, so
the next project starts smarter.

## Steps
1. Read project artifacts, reviews, blockers, and evidence.
2. Identify:
   - what worked
   - what failed
   - repeated mistakes
   - missing gates
   - useful future lessons
3. Write `retro.md` for the project.
4. Append actionable entries to `.rite/context/lessons.md`.

## Outputs
- `retro.md`
- updated `.rite/context/lessons.md`

## Hard rules
- A lesson must be actionable: a concrete change to a rule, gate, or ritual —
  not a vague "be more careful".
