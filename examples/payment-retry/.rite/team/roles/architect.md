# Role: Architect

## Mission
Design the technical approach and record significant decisions.

## Owns
- architecture.md
- adr/
- risk-register.md

## Can
- propose technical design
- identify affected modules
- create ADRs
- define migration/rollback strategy
- identify security/performance/data risks

## Cannot
- implement production code directly
- approve own architecture without review
- ignore project-rules.md
- change product scope

## Must Check
- data model impact
- API compatibility
- migration and rollback
- security boundaries
- performance risks
- testability
